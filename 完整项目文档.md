## 🚀 快速开始

### 第一次使用

1. **打开 MINGW64 终端**（不是 UCRT64 或 MSYS2）
2. **安装依赖**：
   ```bash
   cd scripts
   ./安装依赖.sh
   ```
3. **编译驱动**：
   ```bash
   ./编译驱动.sh
   ```
4. **配置防火墙**（Windows）：
   ```cmd
   scripts\配置防火墙.bat
   ```

### 启动游戏

**方式一：使用批处理脚本（推荐）**
```cmd
scripts\启动游戏.bat
scripts\启动Web客户端.bat
```

**方式二：手动启动**
```bash
# 在 MINGW64 终端
cd fluffos/build
./bin/driver.exe ../../mymud/config.cfg

# 另开一个终端启动 Web 服务器
cd mymud/www
python -m http.server 8000 --bind 0.0.0.0
```

### 访问游戏

- **本地访问**：http://localhost:8000
- **局域网访问**：http://你的IP:8000（例如：http://192.168.1.150:8000）
- **Telnet客户端**：telnet localhost 4000

## ⚙️ 环境配置

### 必需软件

1. **MSYS2/MINGW64**
   - 下载：https://www.msys2.org/
   - 安装后打开 "MINGW64" 终端（不是 UCRT64）

2. **编译工具链**
   ```bash
   pacman -S --needed base-devel mingw-w64-x86_64-toolchain
   pacman -S mingw-w64-x86_64-cmake
   pacman -S mingw-w64-x86_64-zlib
   pacman -S mingw-w64-x86_64-libevent
   pacman -S mingw-w64-x86_64-pcre
   ```

3. **Python 3**（用于 Web 服务器）
   - Windows 自带或从 python.org 下载

### 端口配置

- **4000**：Telnet 端口
- **8080**：WebSocket 端口（游戏服务器）
- **8000**：HTTP 端口（Web 客户端）

### 防火墙规则

运行 `scripts\配置防火墙.bat` 自动添加规则，或手动添加：
```cmd
netsh advfirewall firewall add rule name="MUD Telnet" dir=in action=allow protocol=TCP localport=4000
netsh advfirewall firewall add rule name="MUD WebSocket" dir=in action=allow protocol=TCP localport=8080
netsh advfirewall firewall add rule name="MUD Web" dir=in action=allow protocol=TCP localport=8000
```

## 🎮 启动游戏

### 重要提示

1. **必须使用 MINGW64 终端**
2. **驱动必须保持运行**（不要关闭窗口或按 Ctrl+C）
3. **需要同时运行两个服务**：
   - 游戏驱动（端口 8080）
   - Web 服务器（端口 8000）

### 启动步骤

1. **启动游戏驱动**
   ```bash
   cd fluffos/build
   ./bin/driver.exe ../../mymud/config.cfg
   ```
   看到 "Accepting connections" 表示成功

2. **启动 Web 服务器**（另开终端）
   ```bash
   cd mymud/www
   python -m http.server 8000 --bind 0.0.0.0
   ```

3. **打开浏览器**
   - 访问：http://localhost:8000
   - 或：http://你的IP:8000

### 停止游戏

1. 在游戏中输入 `quit` 退出
2. 在驱动终端按 Ctrl+C 停止驱动
3. 在 Web 服务器终端按 Ctrl+C 停止服务器

## 🏠 局域网访问

### 配置步骤

1. **查看本机IP**
   ```cmd
   ipconfig
   ```
   找到 "IPv4 地址"，例如：192.168.1.150

2. **配置防火墙**
   ```cmd
   scripts\配置防火墙.bat
   ```

3. **启动服务**（绑定到 0.0.0.0）
   ```bash
   # 游戏驱动已配置为 0.0.0.0
   cd fluffos/build
   ./bin/driver.exe ../../mymud/config.cfg
   
   # Web 服务器
   cd mymud/www
   python -m http.server 8000 --bind 0.0.0.0
   ```

4. **其他设备访问**
   - 浏览器：http://192.168.1.150:8000
   - Telnet：telnet 192.168.1.150 4000

### 服务器部署

部署到公网服务器（Linux）：

1. **安装依赖**
   ```bash
   # Ubuntu/Debian
   sudo apt-get install build-essential cmake zlib1g-dev libevent-dev libpcre3-dev
   
   # CentOS/RHEL
   sudo yum install gcc-c++ cmake zlib-devel libevent-devel pcre-devel
   ```

2. **编译驱动**
   ```bash
   cd fluffos
   mkdir build && cd build
   cmake ..
   make
   ```

3. **配置反向代理**（Nginx）
   ```nginx
   server {
       listen 80;
       server_name yourdomain.com;
       
       location / {
           root /path/to/mymud/www;
           index index.html;
       }
       
       location /ws {
           proxy_pass http://localhost:8080;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "upgrade";
       }
   }
   ```

4. **使用 systemd 管理服务**
   ```ini
   [Unit]
   Description=FluffOS MUD Game
   After=network.target
   
   [Service]
   Type=simple
   User=muduser
   WorkingDirectory=/path/to/fluffos/build
   ExecStart=/path/to/fluffos/build/bin/driver /path/to/mymud/config.cfg
   Restart=always
   
   [Install]
   WantedBy=multi-user.target
   ```

## ❓ 常见问题

### 编译问题

**Q: 编译时提示找不到库文件？**
A: 确保使用 MINGW64 终端，并已安装所有依赖：
```bash
pacman -S mingw-w64-x86_64-zlib mingw-w64-x86_64-libevent mingw-w64-x86_64-pcre
```

**Q: CMake 配置失败？**
A: 删除 build 目录重新编译：
```bash
cd fluffos
rm -rf build
mkdir build && cd build
cmake ..
make
```

### 运行问题

**Q: 驱动启动后立即退出？**
A: 检查配置文件路径是否正确：
```bash
./bin/driver.exe ../../mymud/config.cfg
```

**Q: 提示端口被占用？**
A: 检查端口是否被其他程序占用：
```cmd
netstat -ano | findstr "4000"
netstat -ano | findstr "8080"
netstat -ano | findstr "8000"
```

### Web客户端问题

**Q: 页面显示空白？**
A: 确保 Web 服务器正在运行：
```bash
cd mymud/www
python -m http.server 8000
```

**Q: 显示旧版本界面？**
A: 强制刷新浏览器缓存：
- Windows: Ctrl + Shift + R
- Mac: Cmd + Shift + R
- 手机：清除浏览器缓存

**Q: 无法连接到游戏服务器？**
A: 检查：
1. 游戏驱动是否正在运行
2. 防火墙是否允许端口 8080
3. 浏览器控制台是否有错误信息

**Q: 显示乱码或菱形符号？**
A: 最新版本已修复此问题，请强制刷新浏览器缓存。

### 局域网访问问题

**Q: 手机无法访问？**
A: 检查：
1. 手机和电脑在同一局域网
2. 防火墙已配置（运行 `scripts\配置防火墙.bat`）
3. 使用正确的IP地址（不是 localhost）

**Q: 连接被拒绝？**
A: 确保服务绑定到 0.0.0.0 而不是 127.0.0.1：
```bash
python -m http.server 8000 --bind 0.0.0.0
```

### 游戏问题

**Q: 输入命令没有反应？**
A: 检查：
1. WebSocket 连接状态（页面右上角）
2. 命令是否正确（输入 `help` 查看帮助）
3. 浏览器控制台是否有错误

**Q: 无法突破境界？**
A: 需要满足条件：
- 经验值达到要求
- 灵力充足
- 使用 `tupo` 命令

**Q: 数据会保存吗？**
A: 会自动保存，保存时机：
- 退出游戏时（`quit` 命令）
- 断开连接时
- 升级时

## 🔧 开发说明

### 添加新房间

1. 在 `mymud/room/` 创建新文件
2. 继承 `ROOM` 对象
3. 实现 `create()` 函数
4. 设置房间属性和出口

示例：
```c
inherit ROOM;

void create() {
    set("short", "神秘洞穴");
    set("long", "这是一个神秘的洞穴，四周漆黑一片。\n");
    
    set("exits", ([
        "out" : "/room/xiuxian/forest",
    ]));
}
```

### 添加新NPC

1. 在 `mymud/npc/` 创建新文件
2. 实现 NPC 行为
3. 在房间中加载 NPC

### 添加新物品

1. 在 `mymud/obj/` 创建新文件
2. 设置物品属性
3. 实现物品功能

### 修改游戏逻辑

主要逻辑在 `mymud/clone/xiuxian_user.c`，包括：
- 命令处理（`process_input()`）
- 修炼系统
- 战斗系统
- 物品系统
