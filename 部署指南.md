## 🖥️ 本地开发环境

### Windows (MINGW64)

#### 1. 安装依赖
```bash
# 打开 MINGW64 终端
cd scripts
./安装依赖.sh
```

#### 2. 编译驱动
```bash
./编译驱动.sh
```

#### 3. 启动服务
```bash
# 方式一：使用批处理脚本
scripts\启动游戏.bat
scripts\启动Web客户端.bat

# 方式二：手动启动
cd fluffos/build
./bin/driver.exe ../../mymud/config.cfg

# 另开终端
cd mymud/www
python -m http.server 8000 --bind 0.0.0.0
```

### Linux / macOS

#### 1. 安装依赖
```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y build-essential cmake git \
    zlib1g-dev libevent-dev libpcre3-dev python3

# CentOS/RHEL
sudo yum install -y gcc-c++ cmake git \
    zlib-devel libevent-devel pcre-devel python3

# macOS
brew install cmake libevent pcre
```

#### 2. 编译驱动
```bash
cd fluffos
mkdir -p build && cd build
cmake ..
make -j$(nproc)
```

#### 3. 启动服务
```bash
# 终端1：启动游戏驱动
cd fluffos/build
./bin/driver ../../mymud/config.cfg

# 终端2：启动 Web 服务器
cd mymud/www
python3 -m http.server 8000 --bind 0.0.0.0
```

## 🏠 局域网部署

### 1. 配置防火墙

#### Windows
```cmd
# 运行脚本自动配置
scripts\配置防火墙.bat

# 或手动添加规则
netsh advfirewall firewall add rule name="MUD Telnet" dir=in action=allow protocol=TCP localport=4000
netsh advfirewall firewall add rule name="MUD WebSocket" dir=in action=allow protocol=TCP localport=8080
netsh advfirewall firewall add rule name="MUD Web" dir=in action=allow protocol=TCP localport=8000
```

#### Linux
```bash
# UFW
sudo ufw allow 4000/tcp
sudo ufw allow 8080/tcp
sudo ufw allow 8000/tcp

# firewalld
sudo firewall-cmd --permanent --add-port=4000/tcp
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload

# iptables
sudo iptables -A INPUT -p tcp --dport 4000 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 8000 -j ACCEPT
```

### 2. 查看本机 IP

#### Windows
```cmd
ipconfig
# 查找 "IPv4 地址"
```

#### Linux/macOS
```bash
# 方法1
ip addr show
# 或
ifconfig

# 方法2：快速查看
hostname -I
```

### 3. 访问游戏

- **本机**：http://localhost:8000
- **局域网其他设备**：http://192.168.1.X:8000（替换为你的IP）

## 🌐 服务器部署

### 方案一：直接部署

#### 1. 准备服务器

```bash
# 更新系统
sudo apt-get update && sudo apt-get upgrade -y

# 安装依赖
sudo apt-get install -y build-essential cmake git \
    zlib1g-dev libevent-dev libpcre3-dev \
    python3 nginx supervisor
```

#### 2. 克隆代码

```bash
cd /opt
sudo git clone <你的仓库地址> mudgame
cd mudgame
sudo chown -R $USER:$USER .
```

#### 3. 编译驱动

```bash
cd fluffos
mkdir -p build && cd build
cmake ..
make -j$(nproc)
```

#### 4. 配置 Supervisor（进程管理）

创建 `/etc/supervisor/conf.d/mudgame.conf`：

```ini
[program:mud-driver]
command=/opt/mudgame/fluffos/build/bin/driver /opt/mudgame/mymud/config.cfg
directory=/opt/mudgame/fluffos/build
user=muduser
autostart=true
autorestart=true
stderr_logfile=/var/log/mud-driver.err.log
stdout_logfile=/var/log/mud-driver.out.log

[program:mud-web]
command=/usr/bin/python3 -m http.server 8000 --bind 127.0.0.1
directory=/opt/mudgame/mymud/www
user=muduser
autostart=true
autorestart=true
stderr_logfile=/var/log/mud-web.err.log
stdout_logfile=/var/log/mud-web.out.log
```

启动服务：
```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start mud-driver
sudo supervisorctl start mud-web
```

#### 5. 配置 Nginx（反向代理）

创建 `/etc/nginx/sites-available/mudgame`：

```nginx
server {
    listen 80;
    server_name yourdomain.com;  # 替换为你的域名

    # Web 客户端静态文件
    location / {
        root /opt/mudgame/mymud/www;
        index index.html;
        try_files $uri $uri/ =404;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400;
    }
}
```

启用站点：
```bash
sudo ln -s /etc/nginx/sites-available/mudgame /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 6. 配置 HTTPS（可选但推荐）

```bash
# 安装 Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d yourdomain.com

# 自动续期
sudo certbot renew --dry-run
```

### 方案二：使用 systemd

创建 `/etc/systemd/system/mud-driver.service`：

```ini
[Unit]
Description=FluffOS MUD Game Driver
After=network.target

[Service]
Type=simple
User=muduser
WorkingDirectory=/opt/mudgame/fluffos/build
ExecStart=/opt/mudgame/fluffos/build/bin/driver /opt/mudgame/mymud/config.cfg
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

创建 `/etc/systemd/system/mud-web.service`：

```ini
[Unit]
Description=MUD Web Server
After=network.target

[Service]
Type=simple
User=muduser
WorkingDirectory=/opt/mudgame/mymud/www
ExecStart=/usr/bin/python3 -m http.server 8000 --bind 127.0.0.1
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable mud-driver mud-web
sudo systemctl start mud-driver mud-web
sudo systemctl status mud-driver mud-web
```

## 🐳 Docker 部署

### 1. 创建 Dockerfile

创建 `Dockerfile`：

```dockerfile
FROM ubuntu:22.04

# 安装依赖
RUN apt-get update && apt-get install -y \
    build-essential \
    cmake \
    git \
    zlib1g-dev \
    libevent-dev \
    libpcre3-dev \
    python3 \
    && rm -rf /var/lib/apt/lists/*

# 创建工作目录
WORKDIR /app

# 复制代码
COPY . .

# 编译驱动
RUN cd fluffos && \
    mkdir -p build && \
    cd build && \
    cmake .. && \
    make -j$(nproc)

# 暴露端口
EXPOSE 4000 8080 8000

# 创建启动脚本
RUN echo '#!/bin/bash\n\
cd /app/fluffos/build && ./bin/driver /app/mymud/config.cfg &\n\
cd /app/mymud/www && python3 -m http.server 8000 --bind 0.0.0.0\n\
' > /start.sh && chmod +x /start.sh

CMD ["/start.sh"]
```

### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  mudgame:
    build: .
    ports:
      - "4000:4000"   # Telnet
      - "8080:8080"   # WebSocket
      - "8000:8000"   # HTTP
    volumes:
      - ./mymud/data:/app/mymud/data
    restart: unless-stopped
    container_name: mudgame

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./mymud/www:/usr/share/nginx/html:ro
    depends_on:
      - mudgame
    restart: unless-stopped
```

### 3. 创建 nginx.conf

```nginx
events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    upstream mudgame {
        server mudgame:8080;
    }

    server {
        listen 80;
        server_name _;

        location / {
            root /usr/share/nginx/html;
            index index.html;
        }

        location /ws {
            proxy_pass http://mudgame;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_read_timeout 86400;
        }
    }
}
```

### 4. 构建和运行

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 🔄 反向代理配置

### Nginx 完整配置

```nginx
# /etc/nginx/sites-available/mudgame

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 安全头
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    # 静态文件
    location / {
        root /opt/mudgame/mymud/www;
        index index.html;
        try_files $uri $uri/ =404;
        
        # 缓存控制
        location ~* \.(html)$ {
            add_header Cache-Control "no-cache, must-revalidate";
        }
        location ~* \.(js|css|png|jpg|jpeg|gif|ico)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
    }

    # 日志
    access_log /var/log/nginx/mudgame.access.log;
    error_log /var/log/nginx/mudgame.error.log;
}
```

### Caddy 配置

创建 `Caddyfile`：

```caddy
yourdomain.com {
    # 自动 HTTPS
    
    # 静态文件
    root * /opt/mudgame/mymud/www
    file_server
    
    # WebSocket 代理
    @websocket {
        path /ws
    }
    reverse_proxy @websocket localhost:8080 {
        header_up Host {host}
        header_up X-Real-IP {remote}
        header_up X-Forwarded-For {remote}
        header_up X-Forwarded-Proto {scheme}
    }
    
    # 日志
    log {
        output file /var/log/caddy/mudgame.log
    }
}
```

### Apache 配置

```apache
<VirtualHost *:80>
    ServerName yourdomain.com
    Redirect permanent / https://yourdomain.com/
</VirtualHost>

<VirtualHost *:443>
    ServerName yourdomain.com
    
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/yourdomain.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/yourdomain.com/privkey.pem
    
    DocumentRoot /opt/mudgame/mymud/www
    
    <Directory /opt/mudgame/mymud/www>
        Options -Indexes +FollowSymLinks
        AllowOverride None
        Require all granted
    </Directory>
    
    # WebSocket 代理
    ProxyRequests Off
    ProxyPreserveHost On
    
    <Location /ws>
        ProxyPass ws://127.0.0.1:8080/
        ProxyPassReverse ws://127.0.0.1:8080/
    </Location>
    
    ErrorLog ${APACHE_LOG_DIR}/mudgame-error.log
    CustomLog ${APACHE_LOG_DIR}/mudgame-access.log combined
</VirtualHost>
```

## 🔒 安全建议

### 1. 创建专用用户

```bash
sudo useradd -r -s /bin/bash -d /opt/mudgame muduser
sudo chown -R muduser:muduser /opt/mudgame
```

### 2. 限制文件权限

```bash
chmod 755 /opt/mudgame
chmod 700 /opt/mudgame/mymud/data
chmod 600 /opt/mudgame/mymud/config.cfg
```

### 3. 配置防火墙

```bash
# 只开放必要端口
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 4000/tcp  # Telnet（可选）
sudo ufw enable
```

### 4. 使用 fail2ban

```bash
sudo apt-get install fail2ban

# 配置 /etc/fail2ban/jail.local
[DEFAULT]
bantime = 3600
findtime = 600
maxretry = 5
```

## 🚀 性能优化

### 1. 系统优化

```bash
# 增加文件描述符限制
echo "* soft nofile 65535" | sudo tee -a /etc/security/limits.conf
echo "* hard nofile 65535" | sudo tee -a /etc/security/limits.conf

# 优化网络参数
sudo sysctl -w net.core.somaxconn=1024
sudo sysctl -w net.ipv4.tcp_max_syn_backlog=2048
```

### 2. Nginx 优化

```nginx
worker_processes auto;
worker_connections 2048;

gzip on;
gzip_types text/plain text/css application/json application/javascript;
gzip_min_length 1000;
```

### 3. 使用 CDN

将静态资源（JS、CSS、图片）托管到 CDN，减轻服务器压力。
